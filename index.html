一、html 和 css 部分

1. 常见的块状元素和行内元素？

答：块状元素（div p h1 h2 h3 h4 form ul），行内元素（a b br i span input select）

2. css引入的方式有哪些? link和@import的区别是?

内联 内嵌 外链 导入 区别 ：同时加载 前者无兼容性，后者CSS2.1以下浏览器不支持 Link 支持使用javascript改变样式，后者不可

3. css选择符的优先级别？

答：行内>id>class>直接选中>通配符*>间接选中

4. 浏览器内核？

 答：
 IE: trident内核
Firefox：gecko内核
 Safari：webkit内核
Opera：以前是presto内核，Opera现已改用Google Chrome的Blink内核
Chrome：Blink(基于webkit，Google与Opera Software共同开发)

5. 超链接四个状态书写的正确顺序是？

答：link visited hover active

6. 标签上title与alt属性的区别是什么？

答：Alt 当图片不显示时替代显示的文字，Title 为该属性提供信息当鼠标移到图片上面是显示的文字

7. 描述css reset的作用及常用的解决方案？

答：（1）作用，reset重置浏览器因品种不同、版本不同产生的的css默认属性区别 （2）解决方法，手动重置、引入normalize.css或引入respond.js文件等；
表示几层含义？
答：（1）当前文件为html5文件 （2）当前文件遵循浏览器标准模式，不会按照不同浏览器私有标准执行

8. 浏览器标准模式和怪异模式之间的区别是什么?

答：盒子模型的渲染模式的不同，使用 window.top.document.compatMode 可显示为什么模式

9. 如何对网站的文件和资源进行优化?

答：文件合并（如精灵图）、文件最小化/文件压缩、使用CDN托管、使用缓存

10. 清除浮动最好的一种方法？

 答：
 .clearfix:before,.clearfix:after{content:""; display:table;} 
.clearfix:after{clear:both;} 
.clearfix{*zoom:1;/*IE/7/6*/}

11. 引入外部字体的方法？

 答：
 @font-face{

    font-family:'YourWebFontName';

    src:url('YourWebFontName.eot'); /\* IE9 Compat Modes \*/

    src:url('YourWebFontName.eot?#iefix') format('embedded-opentype'), /\* IE6-IE8 \*/

    url('YourWebFontName.woff') format('woff'), /\* Modern Browsers \*/
    url('YourWebFontName.ttf')  format('truetype'), /\* Safari, Android, iOS \*/

    url('YourWebFontName.svg#YourWebFontName') format('svg'); /\* Legacy iOS \*/
}

12. 简述一下src与href的区别

 答：href 是指向网络资源所在位置，建立和当前元素（锚点）或当前文档（链接）之间的链接，用于超链接。
src 是指向外部资源的位置，指向的内容将会嵌入到文档中当前标签所在位置；
在请求src资源时会将其指向的资源下载并应用到文档内，例如js脚本，img图片和frame等元素。
当浏览器解析到该元素时，会暂停其他资源的下载和处理，直到将该资源加载、编译、执行完毕，
图片和框架等元素也如此，类似于将所指向资源嵌入当前标签内。
这也是为什么将js脚本放在底部而不是头部。

13. 什么是CSS Hack？

答：
    // 1、条件Hack
  <!--[if IE]>
     <style>
           .test{color:red;}
     </style>
  <![endif]-->
  // 2、属性Hack
   .test{
   color:#090\9; /* For IE8+ */
   *color:#f00;  /* For IE7 and earlier */
   _color:#ff0;  /* For IE6 and earlier */
   }
  // 3、选择符Hack
   * html .test{color:#090;}       /* For IE6 and earlier */
   * + html .test{color:#ff0;}     /* For IE7 */

14. 什么叫优雅降级和渐进增强？

 答：
渐进增强 progressive enhancement：针对低版本浏览器进行构建页面，保证最基本的功能，
然后再针对高级浏览器进行效果、交互等改进和追加功能达到更好的用户体验。
优雅降级 graceful degradation：一开始就构建完整的功能，然后再针对低版本浏览器进行兼容。
区别：
a.    优雅降级是从复杂的现状开始，并试图减少用户体验的供给
b.    渐进增强则是从一个非常基础的，能够起作用的版本开始，并不断扩充，以适应未来环境的需要
c.    降级（功能衰减）意味着往回看；而渐进增强则意味着朝前看，同时保证其根基处于安全地带
